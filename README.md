# 🏠 Project Design

Веб-приложение для дизайна интерьера с системой аутентификации пользователей и квизом для подбора стиля.

## ✨ Возможности

- 🔐 **Система аутентификации** - регистрация, вход, управление сессиями
- 🎨 **Интерактивный квиз** - подбор дизайна интерьера по предпочтениям
- 👨‍💼 **Админ панель** - управление заявками и пользователями
- 📱 **Адаптивный дизайн** - работает на всех устройствах
- 🔄 **Auto keep-alive** - предотвращение "сна" на Render.com

## 🛠️ Технологический стек

### Backend
- **FastAPI** - современный Python фреймворк
- **SQLite** - база данных
- **Pydantic** - валидация данных
- **Uvicorn** - ASGI сервер

### Frontend
- **Vanilla JavaScript** - без фреймворков
- **CSS3** - стилизация с анимациями
- **HTML5** - семантическая разметка

### Деплой
- **Render.com** - хостинг
- **Python 3.11** - среда выполнения
- **Auto keep-alive** - поддержка активности

## 🚀 Быстрый старт

### Локальный запуск

1. **Клонируйте репозиторий**
   ```bash
   git clone <repository-url>
   cd project-design
   ```

2. **Установите зависимости**
   ```bash
   pip install -r requirements.txt
   ```

3. **Запустите приложение**
   ```bash
   # Windows
   start.bat
   
   # Linux/Mac
   uvicorn server:app --host 0.0.0.0 --port 5000
   ```

4. **Откройте в браузере**
   ```
   http://localhost:5000
   ```

### Деплой на Render.com

1. **Подключите репозиторий к Render.com**
2. **Создайте Web Service**
3. **Используйте настройки из `render.yaml`**
4. **Установите переменные окружения:**
   - `APP_URL=https://your-app.onrender.com`

## 📋 API Эндпоинты

### Аутентификация
- `POST /api/auth/check-phone` - проверка телефона
- `POST /api/auth/login` - вход
- `POST /api/auth/register` - регистрация
- `POST /api/auth/logout` - выход
- `GET /api/auth/current-user` - текущий пользователь

### Квиз
- `POST /api/quiz/submit` - отправка заявки
- `GET /api/quiz/submissions` - список заявок

### Админ панель
- `POST /api/auth/admin-login` - вход в админку
- `GET /api/admin/submissions` - заявки для админа
- `PUT /api/admin/submissions/{id}` - обновление статуса

### Сессии
- `POST /api/session/data` - данные сессии
- `POST /api/session/save` - сохранение сессии

## 🎯 Функционал

### Для пользователей
- **Регистрация** по номеру телефона
- **Вход** в личный кабинет
- **Прохождение квиза** для подбора дизайна
- **Отправка заявки** на консультацию

### Для администраторов
- **Доступ** по админ коду (`ADMIN123`)
- **Просмотр** всех заявок
- **Управление** статусами заявок
- **Фильтрация** и сортировка

## 🔧 Конфигурация

### Переменные окружения
- `APP_URL` - URL приложения для keep-alive пингов
- `PORT` - порт сервера (автоматически на Render)

### Настройки keep-alive
- **Интервал**: 5 минут (300 секунд)
- **Автоматический запуск**: при старте приложения
- **Логирование**: статус пингов в консоли

## 🐛 Устранение проблем

### Ошибка сборки pydantic-core
**Проблема**: Несовместимость Python 3.14 с pydantic
**Решение**: Используйте Python 3.11 в `render.yaml`

### Приложение "засыпает" на Render
**Проблема**: Неактивность на free плане
**Решение**: Встроенный keep-alive отправляет ping каждые 5 минут

### CORS ошибки
**Проблема**: Запросы с других доменов
**Решение**: Настроены CORS для localhost и CDN
